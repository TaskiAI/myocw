Lecture Notes

Factorization into A=LUA = LUA=LU

One goal of today's lecture is to understand Gaussian elimination in terms of matrices; to find a matrix LLL such that A=LUA = LUA=LU. We start with some useful facts about matrix multiplication.

Inverse of a product

The inverse of a matrix product ABABAB is B−1A−1B^{-1}A^{-1}B−1A−1.

Transpose of a product

We obtain the transpose of a matrix by exchanging its rows and columns. In other words, the entry in row iii column jjj of AAA is the entry in row jjj column iii of ATA^TAT.

The transpose of a matrix product ABABAB is BTATB^TA^TBTAT. For any invertible matrix AAA, the inverse of ATA^TAT is (A−1)T(A^{-1})^T(A−1)T.

A=LUA = LUA=LU

We've seen how to use elimination to convert a suitable matrix AAA into an upper triangular matrix UUU. This leads to the factorization A=LUA = LUA=LU, which is very helpful in understanding the matrix AAA.

Recall that (when there are no row exchanges) we can describe the elimination of the entries of matrix AAA in terms of multiplication by a succession of elimination matrices EijE_{ij}Eij​, so that A→E21A→E31E21A→⋯→UA \to E_{21}A \to E_{31}E_{21}A \to \dots \to UA→E21​A→E31​E21​A→⋯→U. How expensive is elimination?

Some applications require inverting very large matrices. This is done using a computer, of course. How hard will the computer have to work? How long will it take?

When using elimination to find the factorization A=LUA = LUA=LU we just saw that we can build LLL as we go by keeping track of row subtractions. We have to remember LLL and (the matrix which will become) UUU; we don't have to store AAA or EijE_{ij}Eij​ in the computer's memory.

How many operations does the computer perform during the elimination process for an n×nn \times nn×n matrix? A typical operation is to multiply one row and then subtract it from another, which requires on the order of nnn operations. There are nnn rows, so the total number of operations used in eliminating entries in the first column is about n2n^2n2. The second row and column are shorter; that product costs about (n−1)2(n-1)^2(n−1)2 operations, and so on. The total number of operations needed to factor A into LU is on the order of n3n^3n3:

12+22+⋯+(n−1)2+n2=∑i=1ni2≈∫0nx2 dx=13n3.1^2 + 2^2 + \dots + (n-1)^2 + n^2 = \sum_{i=1}^{n} i^2 \approx \int_0^n x^2 \,dx = \frac{1}{3}n^3.12+22+⋯+(n−1)2+n2=i=1∑n​i2≈∫0n​x2dx=31​n3.

While we're factoring AAA we're also operating on bbb. That costs about n2n^2n2 operations, which is hardly worth counting compared to 13n3\frac{1}{3}n^331​n3.

Row exchanges

What if there are row exchanges? In other words, what happens if there's a zero in a pivot position?

To swap two rows, we multiply on the left by a permutation matrix. For example,

P12=[010100001]P_{12} = \begin{bmatrix} 0 & 1 & 0 \\ 1 & 0 & 0 \\ 0 & 0 & 1 \end{bmatrix}P12​=​010​100​001​​

swaps the first and second rows of a 3×33 \times 33×3 matrix. The inverse of any permutation matrix P is P−1=PTP^{-1} = P^TP−1=PT.

There are n!n!n! different ways to permute the rows of an n×nn \times nn×n matrix (including the permutation that leaves all rows fixed) so there are n!n!n! permutation matrices. These matrices form a multiplicative group.
