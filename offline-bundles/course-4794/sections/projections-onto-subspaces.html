Projections onto subspaces

Projections

If we have a vector bbb and a line determined by a vector aaa, how do we find the point on the line that is closest to bbb?

![Diagram showing a vector b, a line determined by vector a, and the projection p of b onto the line. The vector b extends from the origin. The line passes through the origin along the direction of a. The point p is on the line, and a perpendicular line connects b to p.](figure_1_projection_onto_a_line.png)

Figure 1: The point closest to bbb on the line determined by aaa.

We can see from Figure 1 that this closest point ppp is at the intersection formed by a line through bbb that is orthogonal to aaa. If we think of ppp as an approximation of bbb, then the length of e=b−pe = b - pe=b−p is the error in that approximation.

We could try to find ppp using trigonometry or calculus, but it's easier to use linear algebra. Since ppp lies on the line through aaa, we know p=xap = xap=xa for some number xxx. We also know that aaa is perpendicular to e=b−xae = b - xae=b−xa:

aT(b−xa)=0a^T (b - xa) = 0aT(b−xa)=0
aTb−xaTa=0a^T b - x a^T a = 0aTb−xaTa=0
xaTa=aTbx a^T a = a^T bxaTa=aTb
x=aTbaTax = \frac{a^T b}{a^T a}x=aTaaTb​

And p=ax=aaTbaTap = ax = a \frac{a^T b}{a^T a}p=ax=aaTaaTb​. Doubling bbb doubles ppp. Doubling aaa does not affect ppp.

Projection matrix

We'd like to write this projection in terms of a projection matrix PPP: p=Pbp = Pbp=Pb.

Given p=aaTbaTap = a \frac{a^T b}{a^T a}p=aaTaaTb​, we can rewrite it as:

p=aaTaTabp = \frac{a a^T}{a^T a} bp=aTaaaT​b

so the matrix is:

P=aaTaTaP = \frac{a a^T}{a^T a}P=aTaaaT​

Note that aaTa a^TaaT is a three by three matrix, not a number; matrix multiplication is not commutative.

The column space of PPP is spanned by aaa because for any bbb, PbPbPb lies on the line determined by aaa. The rank of PPP is 1. PPP is symmetric. P2b=PbP^2b = PbP2b=Pb because the projection of a vector already on the line through aaa is just that vector. In general, projection matrices have the properties:

PT=PandP2=PP^T = P \quad \text{and} \quad P^2 = PPT=PandP2=P

Why project?

As we know, the equation Ax=bAx = bAx=b may have no solution. The vector AxAxAx is always in the column space of AAA, and bbb is unlikely to be in the column space. So, we project bbb onto a vector ppp in the column space of AAA and solve Ax^=pA\hat{x} = pAx^=p.

Projection in higher dimensions

In R3\mathbb{R}^3R3, how do we project a vector bbb onto the closest point ppp in a plane? If a1a_1a1​ and a2a_2a2​ form a basis for the plane, then that plane is the column space of the matrix A=[a1a2]A = \begin{bmatrix} a_1 & a_2 \end{bmatrix}A=[a1​​a2​​].

We know that p=x^1a1+x^2a2=Ax^p = \hat{x}_1 a_1 + \hat{x}_2 a_2 = A\hat{x}p=x^1​a1​+x^2​a2​=Ax^. We want to find x^\hat{x}x^. There are many ways to show that e=b−p=b−Ax^e = b - p = b - A\hat{x}e=b−p=b−Ax^ is orthogonal to the plane we're projecting onto, after which we can use the fact that eee is perpendicular to a1a_1a1​ and a2a_2a2​:

a1T(b−Ax^)=0anda2T(b−Ax^)=0a_1^T (b - A\hat{x}) = 0 \quad \text{and} \quad a_2^T (b - A\hat{x}) = 0a1T​(b−Ax^)=0anda2T​(b−Ax^)=0

In matrix form, AT(b−Ax^)=0A^T (b - A\hat{x}) = 0AT(b−Ax^)=0. When we were projecting onto a line, AAA only had one column and so this equation looked like: aT(b−xa)=0a^T (b - xa) = 0aT(b−xa)=0.

Note that e=b−Ax^e = b - A\hat{x}e=b−Ax^ is in the nullspace of ATA^TAT and so is in the left nullspace of AAA. We know that everything in the left nullspace of AAA is perpendicular to the column space of AAA, so this is another confirmation that our calculations are correct.

We can rewrite the equation AT(b−Ax^)=0A^T (b - A\hat{x}) = 0AT(b−Ax^)=0 as:

ATAx^=ATbA^T A \hat{x} = A^T bATAx^=ATb

When projecting onto a line, ATAA^T AATA was just a number; now it is a square matrix. So instead of dividing by aTaa^T aaTa we now have to multiply by (ATA)−1(A^T A)^{-1}(ATA)−1. In nnn dimensions,

x^=(ATA)−1ATb\hat{x} = (A^T A)^{-1} A^T bx^=(ATA)−1ATb
p=Ax^=A(ATA)−1ATbp = A\hat{x} = A(A^T A)^{-1} A^T bp=Ax^=A(ATA)−1ATb
P=A(ATA)−1ATP = A(A^T A)^{-1} A^TP=A(ATA)−1AT

It's tempting to try to simplify these expressions, but if AAA isn't a square matrix we can't say that (ATA)−1=A−1(AT)−1(A^T A)^{-1} = A^{-1} (A^T)^{-1}(ATA)−1=A−1(AT)−1. If AAA does happen to be a square, invertible matrix then its column space is the whole space and contains bbb. In this case PPP is the identity, as we find when we simplify. It is still true that:

PT=PandP2=PP^T = P \quad \text{and} \quad P^2 = PPT=PandP2=P

Figure 2: Three points and a line close to them.

Suppose we're given a collection of data points (t,b)(t,b)(t,b):

{(1,1),(2,2),(3,2)}\{(1,1), (2,2), (3,2)\}{(1,1),(2,2),(3,2)}

and we want to find the closest line b=C+Dtb = C + Dtb=C+Dt to that collection. If the line went through all three points, we'd have:

C+D=1C+D = 1C+D=1
C+2D=2C+2D = 2C+2D=2
C+3D=2C+3D = 2C+3D=2

which is equivalent to:

(111213)(CD)=(122)\begin{pmatrix} 1 & 1 \\ 1 & 2 \\ 1 & 3 \end{pmatrix} \begin{pmatrix} C \\ D \end{pmatrix} = \begin{pmatrix} 1 \\ 2 \\ 2 \end{pmatrix}​111​123​​(CD​)=​122​​

In our example the line does not go through all three points, so this equation is not solvable. Instead we'll solve:
